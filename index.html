PurchaseLog

Inventory, Sales, and Profitability Tracking Application

Overview

PurchaseLog is a full-stack web application designed to streamline inventory management, sales tracking, and profitability analysis. The application provides a complete workflow for managing products, recording transactions, and generating actionable insights into business performance.

Problem Addressed

Businesses often struggle to track inventory across multiple channels, understand the profitability of individual products, and maintain accurate cost history. PurchaseLog consolidates these functions into a single, intuitive interface, reducing manual tracking and enabling data-driven decisions.

Key Features

  • Inventory Management: Create, organize, and manage product groups and individual items
  • Sales Tracking: Record sales transactions and monitor sales performance
  • Purchase Orders: Manage procurement workflows and purchase-order history
  • Profitability Analysis: Calculate and visualize profit margins for products and categories
  • Cost Snapshots: Maintain historical cost data for accurate profitability reporting
  • Analytics: View dashboards and reports on sales trends, inventory levels, and financial performance
  • Data Import/Export: Import inventory data from CSV and export reports for further analysis
  • User-Friendly Interface: Clean, responsive design built with Bootstrap for accessibility across devices

Technical Implementation

Backend: Built with Python and Flask, PurchaseLog uses Flask-SQLAlchemy for database operations and Jinja templates for server-side rendering. SQLite provides a lightweight, portable database solution.

Frontend: The interface combines Bootstrap for responsive layout, vanilla JavaScript for interactivity, and HTML/CSS for semantic structure. All components are designed for usability and accessibility.

Testing: Automated tests using Pytest ensure core functionality remains reliable across updates.

Configuration: Environment-based configuration allows deployment across different environments. Portable Windows startup scripts simplify local development.

Challenges and Lessons

One significant challenge was designing a data model that accurately captures the relationship between purchase costs and sales profitability over time. By implementing cost snapshots, I solved the problem of historical cost tracking, enabling accurate profitability reports even as product costs change.

Another key learning was the importance of clear data validation and user feedback. Early versions lacked comprehensive error handling; refactoring to include detailed validation messages greatly improved usability and reduced data-entry errors.

Technologies Used

  • Python, Flask, Flask-SQLAlchemy
  • SQLite, SQL
  • Jinja templating
  • Bootstrap CSS framework
  • JavaScript (vanilla)
  • HTML5, CSS3
  • Pytest for automated testing
  • Git version control

FullThrottle

3D Kart-Racing Game in Unity

Overview

FullThrottle is a 3D kart-racing game developed in Unity and C#. The project demonstrates comprehensive game development skills including physics simulation, AI behavior, user interface design, and game progression systems.

Development Context

Originally developed as part of game-development coursework, FullThrottle has evolved into a substantial independent project showcasing advanced game mechanics, polished UI systems, and engaging gameplay progression.

Key Features

  • Main Menu System: Polished menu interface with smooth navigation
  • Character Selection: Multiple playable characters with unique attributes
  • Garage System: Vehicle customization and upgrades
  • Track Selection: Multiple racing tracks with distinct themes
  • Three-Lap Racing: Full race mode with lap tracking and timing
  • AI Opponents: Intelligent AI-controlled competitors with varied difficulty levels
  • Drifting Mechanics: Advanced drifting system with boost rewards
  • Boost Pads: Track hazards and speed-boost mechanics
  • Character Progression: Unlockable characters and vehicles
  • Dynamic Obstacles: Spawning obstacles that challenge players throughout races
  • Difficulty Progression: Scaling difficulty as players advance
  • Minimap System: Real-time race position tracking
  • Input Support: Both keyboard and controller input handling

Technical Implementation

Game Engine: Built entirely in Unity with C# scripting. The project demonstrates proficiency in scene management, physics-based gameplay, and event-driven architecture.

Physics & Gameplay: Custom vehicle physics simulate realistic kart behavior including weight distribution, friction, and drift mechanics. AI opponents use pathfinding and decision-making systems to provide competitive challenges.

UI Systems: Comprehensive UI framework handles menus, HUD elements, race timing, and results screens with smooth animations and state management.

Asset Organization: Clean scene structure and prefab system enable rapid level iteration and feature development.

Challenges and Lessons

Implementing convincing AI opponents was a major challenge. I learned to balance AI difficulty with fairness, ensuring the AI doesn't feel omniscient while still providing meaningful competition. Lane-switching and dynamic obstacle avoidance required careful tuning.

Physics simulation for karts required substantial iteration. Early prototypes felt unresponsive; I improved the feel by adjusting acceleration curves, drift friction, and input sensitivity based on playtesting feedback.

Camera work was another learning area. Third-person racing camera placement is subtle but critical for both control responsiveness and visual feedback. Several camera improvements refined the player experience.

Technologies Used

  • Unity game engine
  • C# scripting
  • Physics engine (Unity physics)
  • Scene management
  • UI Canvas system
  • AI and pathfinding
  • Git version control

CampusConnect

University Service Portal UX Design

Overview

CampusConnect is a UX design prototype developed for a Human-Computer Interaction course. The project demonstrates user-centered design methodology, including research, persona development, wireframing, and interactive prototyping.

Design Challenge

Universities offer numerous services and resources to students, but these are often scattered across different websites and portals. CampusConnect addresses this fragmentation by creating a unified, discoverable interface for accessing academic, administrative, residential, and support services.

Design Methodology

  • User Research: Conducted interviews and surveys with students to understand pain points and service-discovery challenges
  • User Personas: Developed personas representing different student segments (first-year students, commuters, graduate students) with distinct needs
  • Information Architecture: Organized services by user goal rather than administrative department, improving intuitive navigation
  • Wireframing: Created wireframes for key user flows including onboarding, service discovery, and quick actions
  • Interactive Prototyping: Built high-fidelity Figma prototype enabling user testing and iteration
  • Accessibility Focus: Prioritized keyboard navigation, color contrast, and screen reader compatibility
  • Iterative Refinement: Incorporated feedback from usability testing to improve navigation and reduce cognitive load

Key Design Features

  • Personalized Dashboard: Quick access to frequently used services based on user role
  • Search-First Interface: Prominent search for users who know what they need
  • Service Categories: Browse by academic, housing, health, technology, and support categories
  • Responsive Design: Works seamlessly on phones, tablets, and desktops
  • Accessibility Standards: WCAG 2.1 AA compliance for color contrast, keyboard navigation, and semantics
  • Clear Call-to-Actions: Distinct action buttons for primary tasks (request help, view status, schedule appointment)

Technologies Used

  • Figma for design and prototyping
  • User research methodologies
  • Interaction design principles
  • Accessibility (WCAG) standards
  • Usability testing and feedback incorporation

Technical Documentation & Knowledge Base

Support Documentation and Internal Knowledge Base

Overview

During my work in IT support services, I created and improved internal documentation and knowledge-base articles to help both users and support staff resolve issues efficiently. This documentation work demonstrates technical writing ability, procedural thinking, and a commitment to knowledge sharing.

Documentation Focus

  • Zoom Sign-In Procedures: Step-by-step guides for accessing Zoom with university credentials, enabling special features, and troubleshooting connection issues
  • Jamf Connect Administrative Privileges: Documentation for granting and managing admin access on macOS devices using Jamf Connect
  • PaperCut Printing: Guides for printing from personal devices to campus printers, including troubleshooting common connection issues

Writing Approach

Effective technical documentation requires understanding the audience. My documentation serves both technical and non-technical users by:

  • Starting with the simplest explanation before diving into advanced troubleshooting
  • Using clear, active language and avoiding jargon where possible
  • Organizing steps in logical sequences with screenshots where helpful
  • Anticipating common questions and mistakes
  • Providing multiple solutions for different scenarios
  • Maintaining consistency in formatting and style

Impact

Well-organized documentation reduces support tickets by enabling self-service resolution. By creating clear, maintainable knowledge-base articles, I helped reduce repeated questions and allowed the support team to focus on complex issues. Updated documentation also ensures consistency in how issues are explained and resolved.

Key Skills Demonstrated

  • Technical writing for diverse audiences
  • Procedural documentation
  • Troubleshooting guidance
  • Information organization and hierarchy
  • User empathy and accessibility
  • Systems thinking (understanding dependencies and workflows)
